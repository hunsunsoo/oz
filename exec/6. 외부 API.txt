< 프로젝트에서 사용하는 외부 서비스 정리 >

1. 카카오 로그인 API 사용

- kakao developer에서 해당 프로젝트를 웹 애플리케이션으로 등록
- restapi key 발급과 redirecturl 작성
- SpringSecurity와 oauth 라이브러리를 사용한 방식이 아닌 post 요청 방식을 이용한 카카오 로그인 구현

2. 카카오 로그인 프로세스

- FrontEnd에서 redirecturl과 restapi key를 사용하여 카카오 로그인을 진행
- 카카오를 통해 인가코드를 발급 받은 후 해당 인가 코드를 backend로 전송
- backend에서 해당 인가코드를 사용하여 카카오에 post요청 진행
- 3에서 진행한 요청으로 엑세스 토큰 및 리프레시 토큰 발급
- 4에서 발급받은 엑세스 토큰을 사용하여 해당 사용자의 정보를 발급받음
- 사용자의 정보를 추출하여 자체적인 회원가입 또는 로그인 로직을 실행 후 자체적인 jwt토큰 발행
- frontend로 반환

3. 구글을 이용한 SMTP 서비스 사용

- 회원 가입 및 비밀번호 변경을 하기 위해 메일 서비스를 사용하였음
- springboot에서 지원하는 smtp 라이브러리 사용
- 구글 계정을 생성하여 계정 코드 발급을 진행
- 계정 코드 springboot에 등록 후 해당 계정을 사용하여 회원 가입 및 비밀번호 변경 진행
